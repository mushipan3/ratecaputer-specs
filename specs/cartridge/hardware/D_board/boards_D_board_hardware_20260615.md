## 2. 概要・目的

D基板はラテカピュータ（CAR-01規格）の日本語入力フロントエンドおよびシステムUI操作を担う専用基板である。ATtiny1604・CH1115 OLED・SKRHAAEジョイスティックの全機能をこの1枚に集約する。

ジョイスティック入力の監視・OLED描画・日本語入力UIをATtiny1604が完結させ、UIAPduino（CH32V003）のCPU負荷を最小化する。日本語入力システムの詳細は `specs/features/japanese_input.md` を、システムメニュー操作の詳細は `features_system_menu_20260612.md` を参照。

## 3. 物理仕様

| 項目 | 仕様 |
|---|---|
| 基板サイズ | 19mm × 27.8mm（フリスクケース内収容・SKRHAAE010配置確認後に確定） |
| 基板厚 | 0.8mm（JLCPCB 2層 FR-4） |
| 実装方式 | 片面実装 |
| パネライズグリッド | 19mm幅（B基板と同幅） |

**ケース内配置方針：**
SKRHAAE010のDIP変換キット（2.54mmピッチ）の実装面積が支配的。ジョイスティック操作面をケース窓から露出させる位置に配置する。OLEDもケース表面の窓から露出させる。

## 4. 回路設計

### 4.1 搭載部品

| 部品 | 型番 | 仕様 | 備考 |
|---|---|---|---|
| MCU | ATtiny1604 | tinyAVR 0-Series・14ピン・16KB Flash・1KB SRAM・20MHz | I2CスレーブかつOLED/ジョイスティック制御 |
| OLED | CH1115 | 0.5インチ・48×88ドット・I2C（0x3C） | 共通I2Cバス接続・描画はATtiny1604が単独担当 |
| ジョイスティック | Alps Alpine SKRHAAE010（秋月 AE-SKRHAAE010-BO） | 4方向＋センタープッシュ・デジタル5ポジション | プルアップ抵抗実装済みDIP変換キット |

### 4.2 ATtiny1604 ピンアサイン

| ピン | 信号名 | 方向 | 接続先 | 備考 |
|---|---|---|---|---|
| PA0 | LOCAL_UPDI_D | INOUT | UIAPduino PD0（エッジパッド経由） | セルフアップデート用。アップデート中はe-ink描画停止/TFT版は拡張デバイスCS解放が必要 |
| PA1 | SDA | INOUT | 共通I2Cバス SDA | UIAPduino・ATtiny202と同一バス。CH1115はD基板内でPA1に接続 |
| PA2 | SCL | IN | 共通I2Cバス SCL | 同上 |
| PA3 | JOY_RIGHT | IN | SKRHAAE010 RIGHT端子 | プルアップ済み・LOW=押下 |
| PA4 | JOY_LEFT | IN | SKRHAAE010 LEFT端子 | 同上 |
| PA5 | JOY_UP | IN | SKRHAAE010 UP端子 | 同上 |
| PA6 | JOY_DOWN | IN | SKRHAAE010 DOWN端子 | 同上 |
| PA7 | JOY_CENTER | IN | SKRHAAE010 CENTER端子 | 同上 |
| PB0〜PB3 | （予備） | — | — | 未使用・将来拡張用 |
| PC0〜PC3 | （予備） | — | — | 未使用・将来拡張用 |
| VCC | VCC_3.3V | — | A基板 Pad3（VCC_3V3_OUT） | |
| GND | GND | — | 共通GND | |

**LOCAL_UPDI_Dの兼用制約：**
UIAPduino PD0はe-ink版ではBUSY_EINK、TFT版ではGPIO4_CSとしても使用する。ATtiny1604アップデート中はe-ink描画コマンドを送らないこと。TFT版は拡張デバイスのCSをネゲートすること。UPDIは1対1接続専用。ATtiny202（A基板）はUIAPduino PD6（LOCAL_UPDI_A）で別途接続。同時アップデート不可。

### 4.3 ジョイスティック仕様

**Alps Alpine SKRHAAE010（秋月電子 AE-SKRHAAE010-BO）**

| 項目 | 仕様 |
|---|---|
| 方式 | デジタル5ポジションスイッチ（アナログ軸なし） |
| 方向数 | 4方向（上・下・左・右）＋センタープッシュ |
| 接点形式 | スイッチ接点（プルアップ時・押下でLOW） |
| 定格 | 50mA / 12V DC |
| 外形 | 7.35×7.5×1.8mm（SMD） |
| 動作寿命 | 200,000サイクル |
| キット付属 | 10kΩプルアップ抵抗実装済み・2.54mmピッチDIP変換基板 |

**ジョイスティック入力ビットフィールド（I2C返答フォーマット・1バイト）：**

```c
#define JOY_UP      0x01  // PA5 LOW
#define JOY_DOWN    0x02  // PA6 LOW
#define JOY_LEFT    0x04  // PA4 LOW
#define JOY_RIGHT   0x08  // PA3 LOW
#define JOY_CENTER  0x10  // PA7 LOW
// bit5〜bit6: 予備（将来拡張）
// bit7: 1=システムコマンド（SYS_CMD）・0=RAWデータ
```

## 6. 外部接続・インターフェース

### 6.1 エッジパッド一覧

D基板のエッジパッド構成は基板サイズ確定後に正式決定する（PENDING #2）。現時点での想定は以下の通り。

| 信号名 | 方向 | 備考 |
|---|---|---|
| VCC_3.3V | 受信 | A基板 Pad3（VCC_3V3_OUT）から供給 |
| GND | 共通 | |
| SDA | 双方向 | 共通I2Cバス（UIAPduino・ATtiny202・CH1115と共有） |
| SCL | 双方向 | 共通I2Cバス |
| LOCAL_UPDI_D | 双方向 | UIAPduino PD0へ接続・セルフアップデート用 |

### 6.2 OLED仕様（CH1115）

CH1115（0.5インチ・48×88ドット）はD基板上に実装し、D基板内でATtiny1604のSDA/SCLと接続する。共通I2Cバスを通じてUIAPduinoからも認識可能（アドレス0x3C）だが、描画はATtiny1604が単独で担当する。

**画面レイアウト（横長・高さ48×幅88ドット）：**

```
┌─────────────────────────────────────────┐  ← 上端
│  確定文字列（9ドット）                     │
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  変換対象ひらがな文（9ドット）              │  区切り対象文字列は反転表示
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  候補文字列群（24ドット）                  │  3行 × 8ドット
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  バッテリー残量（3ドット）                 │
└─────────────────────────────────────────┘  ← 下端
```

**バッテリー残量表示仕様：**

| 項目 | 仕様 |
|---|---|
| 1ブロックのサイズ | 幅8ドット × 高さ3ドット |
| 1ブロックの表す残量 | 10% |
| 満充電（100%）の幅 | 80ドット（8ドット × 10ブロック） |
| 表示位置 | 下端3ドット・左寄せ |
| 端数ブロックの演出 | ゆっくり点滅 |

## 8. 設計根拠メモ

### ATtiny1604採用理由

tinyAVR 0-Seriesとして14ピン・16KB Flash・1KB SRAMを持ち、ジョイスティック5入力・I2C（SDA/SCL）・LOCAL_UPDI_Dを全て収容できる最小構成。ATtiny202（A基板・2KB Flash）と異なりOLED描画・日本語入力UI・システムメニューUIを担うため十分なFlash容量が必要であり、ATtiny1604が適切。

### SKRHAAE010採用理由

Alps Alpine製デジタル5ポジションスイッチ。アナログ軸なしのためADC不要・デジタルGPIOのみで制御できる。秋月電子でプルアップ抵抗実装済みDIP変換キット（AE-SKRHAAE010-BO）として入手でき、実装・配線が容易。動作寿命200,000サイクルはラテカピュータの想定使用年数に対して十分。

### LOCAL_UPDI_D接続先（PD0）

旧仕様（Rev.3.1）ではUIAPduino PD6に接続していたが、PD0/PD6のUPDI割り当て入れ替えによりPD0に変更した。PD0はe-ink版ではBUSY_EINK・TFT版ではGPIO4_CSと兼用であり、ATtiny1604アップデート中はそれぞれ描画停止・CS解放が必要。

### OLEDをATtiny1604が単独管理する理由

UIAPduino（CH32V003）のCPU負荷を最小化するため。日本語入力候補表示・システムメニューUIともにATtiny1604側で完結させることで、UIAPduinoはFM音源・SPI Flash・ゲームロジックに専念できる。
