## 1. 概要

本書はD基板搭載ATtiny1604のファームウェア仕様を定める。ATtiny1604はUIAPduino（CH32V003）のI2Cスレーブとして動作し、ジョイスティック入力・OLED描画・スリープ管理を担う。日本語入力システムおよびシステムメニューを含むシステム機能全般との協調動作の詳細は `specs/features/japanese_input.md` および `specs/features/system_menu.md` を参照。

---

## 2. 動作モード

| モード | 条件 | 動作 |
|---|---|---|
| RAWデータモード | 通常時 | ジョイスティック状態を1バイトビットフィールドでI2C Readに即答 |
| **システムメニューモード** | **ジョイスティックセンター長押し（約1秒）検知時** | **OLEDにシステムメニューUI表示・操作結果をSYS_CMDでUIAPduinoへ通知** |
| 日本語入力モード | UIAPduinoから開始通知を受信中 | ひらがな入力UI・OLED候補表示・ユーザ確定操作を全担当 |
| スリープ | UIAPduinoからスリープ指示を受信 | OLED表示オフ・定期スリープサイクル |

**モード排他制御：** 日本語入力モード中はシステムメニューモードへの遷移を禁止する（逆も同様）。ATtiny1604側でモード状態フラグを管理する。

---

## 3. I2Cスレーブ動作

UIAPduino（CH32V003）をマスターとする共通I2Cバス上でスレーブとして動作する。

| コマンド種別 | 方向 | 内容 |
|---|---|---|
| Read（RAWモード） | ATtiny1604→UIAPduino | ジョイスティック状態1バイト（bit7=0） |
| **Write/Read（システムコマンド）** | **ATtiny1604→UIAPduino** | **SYS_CMDバイト（bit7=1）：メニュー開閉・音量変更等。詳細はfeatures/system_menu.md参照** |
| Write（モード通知） | UIAPduino→ATtiny1604 | 日本語入力モード開始・終了・スリープ指示 |
| Write（バッテリー残量） | UIAPduino→ATtiny1604 | UIAPduinoがATtiny202から取得した残量値を転送 |

## 4. ジョイスティック入力処理

SKRHAAE010はデジタルスイッチのためADC不要。ポーリングで読み取る。

```c
uint8_t joystick_read(void) {
    uint8_t state = 0;
    if (!(PORTA.IN & PIN5_bm)) state |= JOY_UP;
    if (!(PORTA.IN & PIN6_bm)) state |= JOY_DOWN;
    if (!(PORTA.IN & PIN4_bm)) state |= JOY_LEFT;
    if (!(PORTA.IN & PIN3_bm)) state |= JOY_RIGHT;
    if (!(PORTA.IN & PIN7_bm)) state |= JOY_CENTER;
    return state;
}
```

**チャタリング除去：** 同一状態が連続2回（約10ms間隔）確認された時点で確定。

## 5. OLED描画

CH1115（I2Cアドレス0x3C）への描画はATtiny1604が単独で担当する。UIAPduinoはOLEDに直接書き込まない。

### 5.1 動作モード別表示内容

| モード | OLED表示内容 |
|---|---|
| 日本語入力モード | 確定文字列・変換対象ひらがな・候補文字列群・バッテリー残量 |
| 通常モード | バッテリー残量＋スクリーンセーバー等の自由演出 |
| スリープ | OLED表示オフ |

### 5.2 バッテリーデータの取得フロー

ATtiny1604はバッテリー残量データをUIAPduino経由で受け取る。ATtiny202に直接アクセスしない。

```
UIAPduino → ATtiny202へReadリクエスト（定期ポーリング）
         ← 残量値1バイトを受信
UIAPduino → ATtiny1604へWrite（残量値を転送）
ATtiny1604 → 次回OLED描画時にバッテリー表示に反映
```

## 6. スリープ制御

```
[通常時]
ATtiny1604: 定期スリープ（数秒おき）
            ウェイクアップ → バッテリー表示更新 → スリープ

[日本語入力モード開始通知受信時]
ATtiny1604: スリープ禁止・フルスピード動作
            ジョイスティック入力監視・OLED描画・候補表示

[日本語入力モード終了通知受信時]
ATtiny1604: 通常時スリープサイクルへ復帰
```

## 7. メモリマップ

| 領域 | アドレス | サイズ | 内容 |
|---|---|---|---|
| 固定フォントデータ | 0x0000〜0x07FF | 2KB | 英数・ひらがな256文字（PROGMEM） |
| プログラム領域 | 0x0800〜0x1FFF | 6KB | I2C制御・ジョイスティック・OLED描画・ウェアレベリング等 |
| 候補データ動的バッファ | 0x2000〜0x3FFF | 8KB | 64B/ページ×128ページ・自己書き換え |
| 内蔵EEPROM | 0x00〜0x7F | 128B | ウェアレベリングカウンタ |
| 内蔵SRAM | 全域 | 1KB | UI操作・通信ワークエリア |

---

## 8. フラッシュ寿命試算

候補データバッファ（8KB・128ページ）への書き込みを完全巡回ラウンドロビンで分散。

| 想定使用頻度 | 推定寿命 |
|---|---|
| 1日20回変換（ヘビーユーズ） | 約22年 |
| EEPROMの同時消耗 | フラッシュ寿命時点でEEPROM定格の10%のみ消費 |
| チップ交換コスト | ATtiny1604は約100円・ソケット化で容易交換 |
