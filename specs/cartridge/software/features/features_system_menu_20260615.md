## 1. 概要と設計方針

本書はラテカピュータ（CAR-01）において、アプリ実行中にユーザーが呼び出せる**システム機能**の設計方針を定める。

日本語入力・音量調整・スピーカー切り替え等の「アプリに依存しない共通操作」を、アプリを中断せずにユーザーが呼び出せる仕組みとして整理する。

### 1.1 設計目標

- アプリ側は「システム機能が呼ばれた」という事実を意識せず、処理完了後に透過的に再開できること
- 計算機資源（GPIO・SPI等）の消費を最小化すること
- UIはATtiny1604とサブディスプレイ（CH1115 OLED）で完結すること
- 共通プログラムまたは2層モジュールとして実装し、アプリ側の実装負担をゼロにすること

### 1.2 本書の対象機能

| 機能 | 概要 |
|---|---|
| システムメニュー呼び出し | ジョイスティック長押し等で呼び出すオーバーレイメニュー |
| 音量調整 | YMF825マスターボリュームレジスタのソフトウェア制御 |
| 日本語入力 | 既存仕様（`features/japanese_input.md`）と統合 |
| （将来）輝度調整 | TFT/e-inkバックライト制御 |
| （将来）アプリ切り替え | アプリ一覧に戻る |

## 2. アーキテクチャ

### 2.1 役割分担

```
ユーザー操作
    │ ジョイスティック（SKRHAAE010）
    ↓
ATtiny1604（D基板）
    │ ・ジョイスティック入力監視（常時）
    │ ・システム機能呼び出し操作を検知
    │ ・サブディスプレイ（CH1115 OLED）でシステムメニューUI表示
    │ ・ユーザー選択結果をUIAPduinoへ通知
    │ I2C
    ↓
UIAPduino（CH32V003）
    │ ・アプリ実行中にATtiny1604からの割り込み通知を受信
    │ ・システム機能に応じたデバイス操作を実行
    │   （YMF825音量変更: SPI / 画面ブランク: SPI等）
    │ ・アプリへの透過的再開を保証
    ↓
各デバイス（YMF825・TFT等）
```

### 2.2 レイヤー位置づけ

システム機能は**共通プログラム（1層）の一部**として実装する。

アプリ（3層）はシステム機能の存在を意識しない。共通プログラムがATtiny1604からの通知を受けてシステム機能を処理し、アプリ側の実行コンテキストに影響を与えずに完了する。

```
┌─────────────────────────────────────┐
│  3層: アプリ（App Area 8KB）         │ ← システム機能を意識しない
├─────────────────────────────────────┤
│  2層: 共通モジュール領域（512B）      │ ← 数学ライブラリ等
├─────────────────────────────────────┤
│  1層: 共通プログラム（5.5KB）        │ ← システム機能はここに実装
│    ・IAPコンテキストスイッチ          │
│    ・キースキャン（RAWモード）        │
│    ・システム機能ディスパッチャ ← NEW │
└─────────────────────────────────────┘
```

## 3. システム機能呼び出しメカニズム

### 3.1 トリガー操作

ATtiny1604はジョイスティック入力を常時監視する。以下の操作をシステム機能呼び出しトリガーとして定義する。

| 操作 | 動作 |
|---|---|
| センタープッシュ長押し（約1秒） | システムメニューを開く |
| （将来拡張） | 追加トリガー |

**設計方針：** アプリが使用する通常操作（単押し・方向入力）と明確に区別できる操作をトリガーとする。長押しはアプリ側が通常の入力として使用しないため選定した。

### 3.2 I2C通知フォーマット

ATtiny1604はシステム機能トリガーを検知した時点で、通常のRAWデータモード返答とは異なるコマンドをUIAPduinoへ送信する。

**コマンドバイト定義（暫定）：**

```c
// RAWデータモード（通常）
// bit0〜4: ジョイスティック状態（既存定義）

// システム機能通知（拡張）
#define SYS_CMD_MENU_OPEN     0x80  // システムメニュー呼び出し
#define SYS_CMD_VOLUME_UP     0x81  // 音量UP（メニュー内操作）
#define SYS_CMD_VOLUME_DOWN   0x82  // 音量DOWN（メニュー内操作）
#define SYS_CMD_MENU_CLOSE    0x8F  // システムメニュー終了・アプリ再開
// 0x80〜0xFF: システム機能予約領域（bit7=1でシステムコマンドと識別）
```

bit7が1の場合をシステムコマンドとして識別し、RAWデータ（bit7=0）と区別する。

### 3.3 UIAPduino側の処理

UIAPduinoの共通プログラムはI2Cポーリング（またはI2C割り込み）でATtiny1604からの通知を受け取る。システムコマンド受信時の処理フロー：

```
ATtiny1604からSYS_CMD_MENU_OPEN受信
    ↓
共通プログラムのシステム機能ディスパッチャが起動
    ↓
アプリ（App Area）の実行を一時停止
※ IAPコンテキストスイッチは行わない（SRAM退避不要・軽量処理のため）
    ↓
ATtiny1604がOLEDにシステムメニューUIを表示
UIAPduinoはATtiny1604からの操作結果通知を待機
    ↓
ユーザーが操作・選択（ATtiny1604が担当）
    ↓
ATtiny1604がSYS_CMD_VOLUME_UP/DOWN等を送信
UIAPduinoが対応デバイスを操作（YMF825 SPIレジスタ更新等）
    ↓
SYS_CMD_MENU_CLOSEを受信
    ↓
アプリ実行を再開
```

## 4. 音量調整

### 4.1 制御方式

YMF825のマスターボリュームレジスタ（アドレス0x19）をUIAPduinoがSPI経由で書き換える。

```
UIAPduino → SPI（CS_YMF=PD2） → YMF825 Reg0x19
```

**追加部品・GPIO不要。** PAM8302AのSDピン（R5でAMP_5.0Vにプルアップ）はソフトウェア制御しない。

### 4.2 音量レベル定義

| レベル | レジスタ値（0x19） | 概要 |
|---|---|---|
| 0（無音） | 0x00 | ミュート |
| 1〜7 | TBD | 段階調整（7段階推奨） |
| 8（最大） | 0x1F（またはデフォルト） | 最大音量 |

実機調整後に確定する。

### 4.3 スピーカーミュート

スピーカーへのケーブルに物理スイッチを設ける方式を採用する。ソフトウェア・GPIOによる制御は行わない。ヘッドフォンジャックには常時音声信号が出力されるため、プラグを挿せばヘッドフォン・外部スピーカーいずれも使用可能。

## 5. 日本語入力との統合

日本語入力は既存の仕様（`features/japanese_input.md`）で定義済みのシステム機能である。本書のシステムメニュー機構と以下の点で整合させる。

| 項目 | 日本語入力 | システムメニュー（本書） |
|---|---|---|
| トリガー | アプリからiap_call()でWnnブロックを呼び出し | ジョイスティック長押し |
| ATtiny1604の動作 | 日本語入力モード（専用UI） | システムメニューモード（専用UI） |
| UIAPduinoとのI2C | モード開始/終了通知・候補転送 | システムコマンド通知 |
| アプリの関与 | iap_call()を明示的に呼び出す | 透過的（アプリ側実装不要） |

**モードの排他制御：** 日本語入力モード中はシステムメニューを呼び出せない（または無視する）ものとする。逆も同様。ATtiny1604側でモード状態を管理し、競合を防ぐ。

## 6. ATtiny1604ファームウェアへの影響

D基板ファームウェア仕様書（`boards/D_board/firmware.md`）の動作モードに以下を追加する。

| モード | 条件 | 動作 |
|---|---|---|
| RAWデータモード | 通常時 | ジョイスティック状態を1バイトでI2C返答（既存） |
| **システムメニューモード（NEW）** | **長押し検知時** | **OLEDにメニューUI表示・操作をSYS_CMDでUIAPduinoへ通知** |
| 日本語入力モード | UIAPduinoから開始通知受信中 | ひらがな入力UI（既存） |
| スリープ | UIAPduinoからスリープ指示 | OLED表示オフ（既存） |

## 7. OLED画面レイアウト（システムメニューモード）

CH1115 OLED（48×88ドット）に表示するシステムメニューのレイアウト案。

```
┌─────────────────────────────┐  ← 上端
│  ≡ SYSTEM MENU              │  タイトル行（9ドット）
├─────────────────────────────┤
│  ▶ 音量  [====----] 5/8     │  音量（メニュー項目1）
│    輝度  ─ ─ ─ ─           │  輝度（将来）
│    もどる                   │  アプリ再開
└─────────────────────────────┘  ← 下端
```

- 上下でカーソル移動
- 左右で値変更（音量・輝度）
- センタープッシュで確定・決定
- 長押しまたは「もどる」でメニュー終了
