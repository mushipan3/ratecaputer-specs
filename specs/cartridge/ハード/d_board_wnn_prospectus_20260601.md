## 2. システム構成の全体像（改訂版）

### 2.1 基板構成

CAR-01（FRISK筐体）は以下の基板で構成される。

| 基板 | 搭載部品 | 役割 |
|------|---------|------|
| メイン基板（C基板） | UIAPduino（CH32V003）・SPI Flash・FRAM・YMF825 | メインMCU・ストレージ・音源 |
| A基板 | ATtiny202・電源IC群（Buck DCDC・MT3608・P-ch MOSFET） | 電源管理・電源シーケンス制御 |
| **D基板（新設）** | **ATtiny1604・CH1115 OLED・SKRHAAE010ジョイスティック** | **日本語入力フロントエンド** |

### 2.2 I2Cバストポロジー（改訂版）

```
UIAPduino（CH32V003）
    │  I2Cマスター
    │  SDA / SCL（PC1 / PC2）
    ├──────────────────────────────┐
    │                              │
ATtiny202（A基板）           ATtiny1604（D基板）
I2Cスレーブ                  I2Cスレーブ
アドレス: TBD                 アドレス: TBD
    │                              │
    └──────────┬───────────────────┘
               │ 同一I2Cバス
           CH1115 OLED（D基板搭載）
           I2Cスレーブ（アドレス: 0x3C）
           0.5インチ 48×88ドット
```

**マスター:** UIAPduino（CH32V003）1台のみ
**スレーブ:** ATtiny202・ATtiny1604・CH1115 OLEDの3デバイス

Rev.2.0および旧A基板仕様書に記載されていた「ATtiny202をマスターとする構成」は本Rev.3.0で廃止する。UIAPduinoを唯一のマスターとするシンプルなシングルマスター構成に統一する。

## 3. A基板 変更仕様

### 3.1 OLED変更

| 項目 | 旧仕様（Rev.2.0以前） | 新仕様（Rev.3.0） |
|------|---------------------|-----------------|
| ドライバIC | SSD1306 | CH1115 |
| サイズ | 0.42インチ | 0.5インチ |
| 解像度 | 72×40ドット | 48×88ドット |
| 実装基板 | A基板 | **D基板**（A基板から移管） |
| インターフェース | I2C | I2C |
| I2Cアドレス | 0x3C | 0x3C |

A基板からOLEDを撤去し、OLEDはD基板に1枚のみ搭載する。A基板はOLEDを持たない。

### 3.2 I2Cトポロジー変更

旧A基板仕様書に記載されていた「ATtiny202を常時I2Cマスターとする設計」を廃止する。ATtiny202はI2Cスレーブとして動作し、UIAPduinoからのRead要求に対してバッテリー残量・電源状態を返す構成に変更する。

| 項目 | 旧仕様 | 新仕様 |
|------|--------|--------|
| ATtiny202のI2C役割 | マスター | **スレーブ** |
| OLED描画の担当 | ATtiny202が自律描画 | **ATtiny1604が描画** |
| バッテリー残量の通知 | ATtiny202がUIAPduinoへ送信（マスター送信） | **UIAPduinoがATtiny202へReadリクエスト** |

### 3.3 ATtiny202 ピンアサイン（変更後）

```
              ATtiny202 (SOIC-8)
           ┌───────┴───────┐
  VBAT_IN ─┤ 1. VCC   GND 8 ├─── GND_IN
LtcBus_SYS─┤ 2. PA1   PA3 7 ├─── MOSFETゲート制御（10kΩでVBATへPU）
 _INT/SPK  │               │
  SLV_SDA ─┤ 3. PA2   PA0 6 ├─── LOCAL_UPDI（基板内完結）
  SLV_SCL ─┤ 4. PA4   PA5 5 ├─── タクトSW ＆ バッテリー分圧監視回路
           └───────────────┘
```

PA2/PA4はI2Cスレーブとして機能する（旧設計と物理ピンは同一）。

## 4. D基板ハードウェア仕様

### 4.1 概要

D基板はCAR-01の日本語入力フロントエンドを担う専用基板である。ユーザーの入力操作・OLED表示・ATtiny1604によるI2Cスレーブ化の全機能をこの1枚に集約する。

### 4.2 搭載部品

| 部品 | 型番 | 仕様 | 備考 |
|------|------|------|------|
| MCU | ATtiny1604 | tinyAVR 0-Series・14ピン・16KB Flash・1KB SRAM・20MHz | I2CスレーブかつOLED/ジョイスティック制御 |
| OLED | CH1115 | 0.5インチ・48×88ドット・I2C（0x3C） | ATtiny202・UIAPduinoと共有 |
| ジョイスティック | Alps Alpine SKRHAAE010（秋月 AE-SKRHAAE010-BO） | 4方向＋センタープッシュ・デジタル5ポジション・50mA/12V DC | プルアップ抵抗（10kΩ）実装済みDIP変換キット |

### 4.3 ATtiny1604 ピンアサイン

| ピン | 信号名 | 方向 | 接続先 | 備考 |
|------|--------|------|--------|------|
| PA1 | SDA | INOUT | I2Cバス共通 | UIAPduino・ATtiny202・CH1115と同一バス |
| PA2 | SCL | IN | I2Cバス共通 | 同上 |
| PA3 | JOY_RIGHT | IN | SKRHAAE010 RIGHT端子 | プルアップ済み・LOW=押下 |
| PA4 | JOY_LEFT | IN | SKRHAAE010 LEFT端子 | 同上 |
| PA5 | JOY_UP | IN | SKRHAAE010 UP端子 | 同上 |
| PA6 | JOY_DOWN | IN | SKRHAAE010 DOWN端子 | 同上 |
| PA7 | JOY_CENTER | IN | SKRHAAE010 CENTER端子 | 同上 |
| PB0〜PB3 | （予備） | — | — | 未使用・将来拡張用 |
| PC0〜PC3 | （予備） | — | — | 未使用・将来拡張用 |
| VCC | 3.3V | — | VCC_3V3_OUT（A基板から供給） | |
| GND | GND | — | GND共通 | |

### 4.4 ジョイスティック仕様詳細

**Alps Alpine SKRHAAE010（秋月電子 AE-SKRHAAE010-BO）**

| 項目 | 仕様 |
|------|------|
| 方式 | デジタル5ポジションスイッチ（アナログ軸なし） |
| 方向数 | 4方向（上・下・左・右）＋センタープッシュ |
| 接点形式 | スイッチ接点（プルアップ時・押下でLOW） |
| 定格 | 50mA / 12V DC |
| 外形（素子） | 7.35×7.5×1.8mm（SMD） |
| 動作寿命 | 200,000サイクル |

| ATtiny1604との接続 | GPIO 5本（デジタル入力・ADC不要） |

## 5. OLED管理設計

### 5.1 基本方針

CH1115 OLED（I2Cアドレス0x3C）への描画は**ATtiny1604が単独で担当する**。ATtiny202はOLEDに直接書き込まない。

この設計の根拠は以下の通り。

| 検討事項 | 結論 |
|---------|------|
| ATtiny202のFlash（2KB）にOLED描画ライブラリを収容できるか | 困難。U8g2等のライブラリは重すぎる |
| ATtiny202のピンに空きはあるか | ゼロ。全ピン既割り当て済み |
| OLEDはUIAPduino電源オフ時に使用できるか | 不可。OLEDはVCC_3V3_OUTから給電のためUIAPduinoと同時にオフになる |
| セマフォ方式（共有描画）は成立するか | UIAPduinoがいない時にセマフォ置き場が消えるため不成立 |

以上より、ATtiny1604がOLEDの全48ドットを一元管理する集約方式を採用する。

### 5.2 ATtiny1604のOLED動作モード

| モード | 条件 | OLED表示内容 |
|--------|------|------------|
| **日本語入力モード** | UIAPduinoから入力モード開始通知を受信中 | 確定文字列・変換対象ひらがな・候補文字列群・バッテリー残量 |
| **通常モード** | 日本語入力モード以外 | バッテリー残量＋ATtiny1604による自由な演出（スクリーンセーバー等） |
| **スリープ** | UIAPduinoからスリープ指示を受信 | OLED表示オフ |

### 5.3 バッテリーデータの取得フロー

ATtiny202はバッテリー残量の数値（0〜100%・1バイト）を内部変数として保持し、UIAPduinoからのReadリクエストに応答するだけである。

```
定期ポーリング（数秒に1回）:
UIAPduino → ATtiny202へReadリクエスト（I2C）
         ← 残量値1バイトを受信
UIAPduino → ATtiny1604へ残量値をWriteコマンドで転送（I2C）
ATtiny1604 → 次回OLED描画時にバッテリー表示に反映
```

UIAPduinoが中継役となる。ATtiny202・ATtiny1604はスレーブとして応答・受信するだけでよく、双方のファームウェアがシンプルになる。

### 5.4 ATtiny1604のスリープ制御

日本語入力を行わない通常時もATtiny1604はバッテリー表示の定期更新のためにスリープと起動を繰り返す。UIAPduinoからの通知により動作を切り替える。

```
[通常時]
ATtiny1604: 定期スリープ（数秒おき）
            ウェイクアップ → バッテリー表示更新 → スリープ

[日本語入力モード開始通知受信時]
ATtiny1604: スリープ禁止・フルスピード動作
            ジョイスティック入力監視・OLED描画・候補表示

[日本語入力モード終了通知受信時]
ATtiny1604: 通常時スリープサイクルへ復帰
```

UIAPduino → ATtiny1604へのモード通知はI2C Writeコマンドで行う（コマンド定義は別途確定）。

### 5.5 OLED画面レイアウト仕様

OLEDは横長（48×88ドット）で使用する。高さ48ドット・幅88ドット。

#### 高さ方向レイアウト（合計48ドット）

```
┌─────────────────────────────────────────┐  ← 上端
│  確定文字列（9ドット）                     │  ATtiny1604管轄
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  変換対象ひらがな文（9ドット）              │  ATtiny1604管轄
│  ※区切り対象文字列は反転表示              │
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  候補文字列群（24ドット）                  │  ATtiny1604管轄
│  3行 × 8ドット（行間なし）                │
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  バッテリー残量（3ドット）                 │  ATtiny1604管轄（データはUIAPduino経由で取得）
└─────────────────────────────────────────┘  ← 下端

ATtiny1604管轄: 全48ドット（上端から下端まで）
合計: 48ドット
```

#### 候補文字列群エリアの横方向レイアウト（88ドット幅）

```
|◀︎  |←空白1→|　　　候補文字群（82ドット）　　　|←空白1→|  ▶︎|
  3ドット                                              3ドット
  ←── 4ドット ──→                          ←── 4ドット ──→
```

| 区画 | 幅 | 内容 |
|------|-----|------|
| 左矢印 | 3ドット | ◀︎（候補ページ切り替え示唆） |
| 左空白 | 1ドット | 矢印と候補群の間隔 |
| 候補文字群 | 82ドット | 恵梨沙フォント8×8・10文字×3行 |
| 右空白 | 1ドット | 候補群と矢印の間隔 |
| 右矢印 | 3ドット | ▶︎（候補ページ切り替え示唆） |
| **合計** | **88ドット** | |

**候補文字群エリア（82ドット）の文字配置：**
- 恵梨沙フォント 8×8ドット・隙間なし
- 1行あたり 82 ÷ 8 = 10文字（80ドット使用・左右1ドットずつパディング）
- 3行 × 10文字 = 最大30候補を1ページに表示
- 候補が30件を超える場合は矢印◀︎▶︎でページ切り替え

#### 左右矢印（◀︎▶︎）の描画仕様

幅3ドット・高さ24ドット（候補文字列群エリア全体）に縦中央揃えで描画する。各列は水平中央に揃えた縦線で三角形を表現する。

**▶︎（右向き・右端）：**

```
列1（左端）: 高さ24ドット（上端0〜下端23）  ████████████████████████
列2（中央）: 高さ16ドット（上下4ドット空き） ████████████████
列3（右端）: 高さ 8ドット（上下8ドット空き） ████████
```

**◀︎（左向き・左端）：** ▶︎の左右反転

```
列1（左端）: 高さ 8ドット（上下8ドット空き） ████████
列2（中央）: 高さ16ドット（上下4ドット空き） ████████████████
列3（右端）: 高さ24ドット（上端0〜下端23）  ████████████████████████
```

矢印は常時表示する（候補が1ページのみの場合も表示・押下しても無効）。

#### バッテリー残量表示仕様（ATtiny202管轄・下端3ドット）

| 項目 | 仕様 |
|------|------|
| 1ブロックのサイズ | 幅8ドット × 高さ3ドット |
| 1ブロックの表す残量 | 10% |
| 満充電（100%）の幅 | 80ドット（8ドット × 10ブロック） |
| 表示位置 | 下端3ドット・左寄せ（残り8ドットは空白） |
| 現在残量ブロックの演出 | 該当する最右端ブロック（端数ブロック）がゆっくり点滅 |

```
例：残量60%の場合（6ブロック点灯・7ブロック目が点滅）
[████][████][████][████][████][████][████*]
 10%   20%   30%   40%   50%   60%  点滅中
← 80ドット中 56ドット点灯 + 8ドット点滅 →    8ドット空白
```

ATtiny1604が全画面を管理する。バッテリー残量データはUIAPduino経由でATtiny202から取得し、ATtiny1604が描画する。

## 6. ATtiny1604ファームウェア設計（Rev.2.0 第6章の更新）

### 6.1 役割と動作モード（変更なし）

Rev.2.0 第6.1節の内容を引き継ぐ。

| モード | 動作 |
|--------|------|
| RAWデータモード（通常時） | ジョイスティック状態を1バイトビットフィールドでI2C Readに即答 |
| 日本語入力モード | ひらがな入力UI・OLED候補表示・ユーザ確定操作を全担当 |

### 6.2 ジョイスティック入力処理

SKRHAAE010はデジタルスイッチのためADC不要。GPIO割り込みまたはポーリングで読み取る。

```c
// ジョイスティック読み取り（ポーリング方式）
uint8_t joystick_read(void) {
    uint8_t state = 0;
    // PA5〜PA3・PA6・PA7がLOW = 押下（プルアップ済みのため負論理）
    if (!(PORTA.IN & PIN5_bm)) state |= JOY_UP;
    if (!(PORTA.IN & PIN6_bm)) state |= JOY_DOWN;
    if (!(PORTA.IN & PIN4_bm)) state |= JOY_LEFT;
    if (!(PORTA.IN & PIN3_bm)) state |= JOY_RIGHT;
    if (!(PORTA.IN & PIN7_bm)) state |= JOY_CENTER;
    return state;
}
```

**チャタリング除去:** 同一状態が連続2回（約10ms間隔）確認された時点で確定。

### 6.3 OLED描画前のセマフォ取得

```c
// OLED描画前に必ずセマフォを取得する
bool oled_acquire(void) {
    uint8_t flag = semaphore_read();  // UIAPduinoへI2C Read
    if (flag == 0x00) {
        semaphore_write(0x01);        // 自分（ATtiny1604）で取得
        return true;
    }
    if (flag == 0x02) {
        // ATtiny202使用中 → 5ms待機後に強制取得（優先度高）
        _delay_ms(5);
        semaphore_write(0x01);
        return true;
    }
    return false;  // 異常状態
}

void oled_release(void) {
    semaphore_write(0x00);
}
```

### 6.4 メモリマップ（Rev.2.0 第6.3節より変更なし）

| 領域 | アドレス | サイズ | 内容 |
|------|---------|--------|------|
| 固定フォントデータ | 0x0000〜0x07FF | 2KB | 英数・ひらがな256文字（PROGMEM） |
| プログラム領域 | 0x0800〜0x1FFF | 6KB | I2C制御・ジョイスティック・セマフォ・ウェアレベリング等 |

## 7. D基板の物理設計方針

### 7.1 基板サイズ

フリスクケース（62×38×14mm）への収容を前提とする。SKRHAAE010のDIP変換キット（2.54mmピッチ）の実装面積が支配的となるため、サイズはC基板・A基板との配置関係を踏まえて実測後に確定する。

### 7.2 OLED実装

CH1115（0.5インチ・48×88）はD基板に実装し、ユーザーが視認しやすい位置に配置する。OLEDのI2C信号線（SDA/SCL）はD基板上でATtiny1604と接続し、さらにCAR-01共通I2Cバスへ引き出す。

### 7.3 ジョイスティック実装

SKRHAAE010はDIP変換キット使用のためスルーホール実装となる。ジョイスティックの操作面がケース外に露出するか、あるいはケース窓から操作できる位置に配置する。フリスクケースの薄型（14mm）を考慮した実装高さの確認が必要。
