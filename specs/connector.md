# ラテカピュータ 拡張コネクタ共通仕様

**Rev. 1.0 — 2026年5月**
**出典：ラテカピュータ カートリッジ規格 目論見書 Rev.1.0 第1部 1.2節**

---

## 1. 概要

ラテカキーボード・ラテカカートリッジ・ラテカトイに共通して搭載される拡張コネクタの仕様を定める。各デバイスはこのコネクタを介して接続・通信する。

---

## 2. コネクタ形状

| 項目 | 仕様 |
|------|------|
| 形状 | ボックスヘッダ 14ピン 2列（2.54mmピッチ） |
| ピン配置 | 2列×7ピン（上列7ピン・下列7ピン） |
| キーボード側（固定） | ボックスヘッダ オス 14ピン 2列（秋月 110901） |
| カートリッジ側 | ロープロファイルピンソケット 2×7 2.54mm（秋月 102904）カット使用・実装高さ4.5mm |

---

## 3. ピン配置

### 下列（電源・I2C・UART）

| ピン番号 | 信号名 | 説明 |
|---------|--------|------|
| 下列 1 | BAT_IN | バッテリー入力（3.7V・LiPo直結） |
| 下列 2 | GND | グラウンド |
| 下列 3 | 5V_OUT | キーボード側5V出力（XCL103経由） |
| 下列 4 | SDA | I2Cデータ（Qwiicバス共用） |
| 下列 5 | SCL | I2Cクロック（Qwiicバス共用） |
| 下列 6 | UART_TX | UART送信（ESP32-S3空きGPIO） |
| 下列 7 | UART_RX | UART受信（ESP32-S3空きGPIO） |

### 上列（GPIO・SPI）

| ピン番号 | 信号名 | 説明 |
|---------|--------|------|
| 上列 1 | CART_READY | カートリッジ初期化完了通知（全カートリッジ必須実装） |
| 上列 2 | GPIO2 | 汎用GPIO／キーボード連携アプリはKB_NOTIFYとして使用可 |
| 上列 3 | GPIO3 | 汎用GPIO（用途未定） |
| 上列 4 | GPIO4/CS | 汎用GPIO／SPI CS兼用 |
| 上列 5 | SPI_SCK | SPIクロック |
| 上列 6 | SPI_MOSI | SPIデータ（ホスト→デバイス） |
| 上列 7 | SPI_MISO | SPIデータ（デバイス→ホスト） |

---

## 4. 電源設計方針

- 各デバイスはBAT_IN（3.7V）を受けて内部で昇圧・降圧する**独立電源設計**
- キーボード側の5V_OUTは参照用。下位デバイスはこれに依存しない
- LiPo充電は各デバイスのUSB端子経由で行う

---

## 5. ラテカナノの接続

ラテカナノは下列2〜5番の4ピン（GND / 5V / SDA / SCL）のみに接続する。

| ピン番号 | 信号名 | ラテカナノでの用途 |
|---------|--------|-----------------|
| 下列 2 | GND | グラウンド |
| 下列 3 | 5V_OUT | 電源 |
| 下列 4 | SDA | I2C通信 |
| 下列 5 | SCL | I2C通信 |

ラテカナノは内部基板裏面のクロス配線（あみだくじ状）により、外部に対してI2Cデバイスとして擬態する。キーボード・カートリッジ・トイのいずれにも直接挿入可能。

---

## 6. 通信モード

### モード1：スレーブモード（上位デバイスがマスター）

| プロトコル | 実現性 | 速度 | 用途例 |
|-----------|--------|------|--------|
| I2Cスレーブ | ✅ 推奨 | 400kHz / 約50KB/s | 上位からのコマンド受信・状態通知 |
| UARTスレーブ的 | ✅ 可能 | 〜3Mbps / 約300KB/s | 上位からの大量データ受信 |
| SPIスレーブ | △ 低速限定 | 〜4MHz（実用） | タイミング制約が厳しく原則避ける |

### モード2：マスターモード（下位デバイスがマスター）

| プロトコル | 用途例 |
|-----------|--------|
| I2Cマスター | ゲームパッド・センサー・OLEDディスプレイ |
| SPIマスター | 外付けSPI Flash・TFT・その他SPIデバイス |
| UART送信 | ログ出力・外部モジュール制御 |

### モード3：バディモード（対等通信）

| プロトコル | 実現性 | 速度 | 用途例 |
|-----------|--------|------|--------|
| UART 3Mbps + DMA | ✅ 推奨 | 約300KB/s | アプリ・データ転送 |
| UART 115,200〜460,800bps | ✅ 安定 | 11〜46KB/s | コマンド・応答・セーブデータ |

### 信号線とモードの対応

| 信号線 | ピン | 対応モード |
|--------|------|-----------|
| SDA / SCL | 下列4/5 | モード1（I2Cスレーブ）/ モード2（I2Cマスター） |
| TX / RX | 下列6/7 | モード1（UART受信）/ モード3（UART双方向） |
| SCK / MOSI / MISO | 上列5/6/7 | モード2（SPIマスター）/ モード1（SPIスレーブ・低速限定） |

## 7. 接続・起動シーケンス

### CART_READYピン（必須実装）

全デバイスが実装する必須ピン。初期化完了をHIGH出力で通知する。

```
上位デバイスON
  CART_READYピン（上列1）をプルダウンで監視開始
    ↓
下位デバイスON
  ブートローダ起動
  200ms待機（競合状態を回避）
    ↓
  初期化完了
  CART_READYピン → HIGH出力
    ↓
上位デバイスがCART_READY HIGHを検出
  I2C 0x55にアクセスしてハンドシェイク
  通信モード確定・通常動作へ
```

### ハンドシェイクパケット（I2C固定アドレス 0x55）

```c
// 上位デバイス → 下位デバイス（8バイト）
typedef struct {
    uint8_t  magic[2];     // 0xCA, 0x72（識別子）
    uint8_t  kb_version;   // ファームバージョン
    uint8_t  kb_features;  // 機能フラグ（UART対応等）
    uint8_t  reserved[4];
} KbHello;

// 下位デバイス → 上位デバイス（16バイト）
typedef struct {
    uint8_t  magic[2];      // 0xCA, 0x52（識別子）
    uint8_t  cart_id;       // デバイスID
    uint8_t  cart_version;  // ファームバージョン
    uint8_t  disp_type;     // 0x01=e-ink / 0x02=TFT / 0x03=OLED
    uint8_t  mode_flags;    // 対応モードフラグ
    uint8_t  uart_speed;    // 0x01=115200 / 0x02=1M / 0x03=3M
    uint8_t  reserved[9];
} CartResponse;
```

## 8. フレキシブル接続（階層スキップ）

基本接続はキーボード→カートリッジ→トイ→ナノの順だが以下も可能：

- ラテカトイをラテカキーボードに直挿し
- ラテカナノをラテカキーボードまたはラテカカートリッジに直挿し
