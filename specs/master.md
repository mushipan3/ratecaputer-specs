## 1. プロジェクト概要

### 1.1 オリジナル「ラテカピュータ」とは

1979年（昭和54年）、シャープが製造・発売した「PC-2000/PC-2001」。
ラジオ・テレビ・カセットを一体化した「ラテカセ」にコンピュータを融合させた、国内外に他事例のない空前絶後のマルチメディアマシン。月産目標300台に対し初期ロットの約200台を生産したのみで終了した、知る人ぞ知る伝説のガジェット。シャープ電卓部門の4人チームが「合体させよう」という純粋な開発熱だけで作り上げた。

### 1.2 本プロジェクトの目的

ラテカピュータの素体である「ラテカセ」を21世紀の現代デバイスと技術で再解釈し、オリジナルが体現したあらゆるメディア統合の熱意をそのままに、その後の数十年の歴史を可能な限り積み込んだ「究極のジェネリックラテカピュータ」を作り出すプロジェクト。

- **プロジェクト名**：ラテカピュータ
- **ローマ字表記**：Ratecaputer（※LatecaputerではなくRatecaputer）
- **アーキテクチャ**：マトリョーシカ型拡張エコシステム

## 2. シリーズ構成と階層

各デバイスはそれぞれ自立したコンピュータであり、物理的な格納関係はない。拡張コネクタによる接続で連携する。

### 2.1 システム全体図

```
ラテカピュータ本体
（オリジナルラテカセ筐体＋新規コンピュータ部一体）
  ↕ USB / Bluetooth
ラテカキーボード［自立コンピュータ／USB端子あり］
  ↕ 拡張コネクタ（基本接続）/ USB（PC・本体との接続）
ラテカカートリッジ［自立コンピュータ／USB端子あり］
  ↕ 拡張コネクタ（基本接続）
ラテカトイ［自立コンピュータ］
  ↕ 拡張コネクタ
ラテカナノ［自立コンピュータ］
```

### 2.2 フレキシブル接続（階層スキップ）

基本系はキーボード→カートリッジ→トイ→ナノの順だが、以下も検討：
- ラテカトイをラテカキーボードに直挿し
- ラテカナノをラテカキーボードまたはラテカカートリッジに直挿し

### 2.3 開発・データ転送フロー

- ラテカキーボード／ラテカカートリッジはUSB端子を持ち、PCと直結可能
- PC（ラテカピュータ本体内蔵、または一般PC）で開発したプログラム・データをUSB経由で双方向転送
- ラテカトイ／ラテカナノへの転送はラテカキーボードまたはラテカカートリッジが母艦として中継
- ラテカピュータ本体への拡張コネクタ搭載は現時点で可能性低

### 2.4 旧称・新称対応表

| 新称（確定） | 旧称・仮称 |
|------------|-----------|
| ラテカキーボード | キーボード、ホスト |
| ラテカカートリッジ | CAR-01、メインカートリッジ |
| ラテカトイ | Toyガジェット、AVR64DD14ガジェット、1604ガジェット |
| ラテカナノ | Nano-C、ATtiny10ユニット |

## 4. 拡張コネクタ規格（共通）

- **形状**：ボックスヘッダ 14ピン 2列（2.54mmピッチ）
- **規格名**：ラテカカートリッジ規格 Rev. 1.0 / 拡張コネクタ仕様
- **別称**：M5系Cardputer ADV（ラテカ規格1.2）準拠

### 下列ピンアサイン（ラテカナノ接続対象）

```
[ 1: - ] [ 2: GND ] [ 3: 5V_OUT ] [ 4: SDA(I2C) ] [ 5: SCL(I2C) ] ...
```

ラテカナノは下列2〜5番の4ピン（GND / 5V / SDA / SCL）に接続。
キーボード・カートリッジ・トイのいずれにも直接挿入可能。

## 5. 各ユニット仕様

### 5.1 ラテカキーボード（第1層）

| 項目 | 仕様 |
|------|------|
| MCU | Waveshare ESP32-S3 Touch LCD 2" |
| キースキャン | MPC23017（プルアップ） |
| 周辺接続 | Qwiic（I2C） |
| 入力 | アナログジョイスティック |
| オーディオ | Adafruit PCM5122 DAC(#6421) →I2S(IO33/34/35)→ 10kΩデュアルギャングボリューム → Adafruit TPA2016D2アンプ(#1712) → スピーカー |
| 補記 | 追加抵抗・カップリングコンデンサ不要 |

---

### 5.2 ラテカカートリッジ（第2層）

**CAR-01 確定仕様**

| 項目 | 仕様 |
|------|------|
| MCU | CH32V003（UIAPduino環境、48MHz、RV32ECコア） |
| Flash（外部） | 16MB SPI Flash（W25Q128JVSIM、SOIC-8）／アセット・辞書格納 |
| FRAM（外部） | 512KB SPI FRAM（MB85RS4MTYPF-G-BCE1）／セーブ領域 |
| ディスプレイ | 1.3inch TFT液晶（ST7789、240×240px、RGB565） |
| SPI構成 | 1ch（SPI1）にFlash・FRAM・TFT・音源が全接続 |
| 転送方式 | DMA（DMA1_Channel3）によるタイムスライス型パイプライン |

**DMA転送サイクル**

1. CPUフェーズ：FlashからSRAMバッファ（512B）へ吸い出し＋合成処理
2. DMAフェーズ：SRAMからTFTへ24MHz一括転送
3. 並行演算：DMA動作中にCPUが3Dレイキャスティング演算

---

### 5.3 ラテカトイ（第3層）

**旧称：Toyガジェット / AVR64DD14ガジェット**

| 項目 | 仕様 |
|------|------|
| MCU | AVR64DD14（SOIC-14） |
| Flash | 64KB |
| SRAM | 8KB |
| クロック | 24MHz |
| DAC | 10-bit内蔵（PD6ピン）／アナログ音声出力 |
| MVIO | Port C独立駆動／レベルシフタ不要 |
| ディスプレイ | 小型TFT（96×96等）／内蔵SRAMでダブルバッファ |
| 通信 | I2Cスレーブ（TWI0）／ラテカカートリッジ（CH32V003）がマスター |
| 筐体 | おもちゃのミニゲーム機筐体を流用・魔改造 |
| 拡張コネクタ | 共通14ピン拡張コネクタ内蔵 |

**ATtiny1604との比較（選定理由）**

| 項目 | ATtiny1604（旧） | AVR64DD14（採用） |
|------|----------------|-----------------|
| Flash | 16KB | 64KB（4倍） |
| SRAM | 1KB | 8KB（8倍） |
| クロック | 20MHz | 24MHz |
| MVIO | なし | あり |
| DAC | なし | 10-bit内蔵 |

**サウンド構成**

- 16kHzサンプリング、ハードウェアタイマー割り込み＋音声ダブルバッファ
- 矩形波ブザーではなく本物のアナログ波形出力（PCMボイス・ノイズ・FMライク3重奏）

---

### 5.4 ラテカナノ（第4層）

**旧称：Nano-C / ATtiny10ユニット**

| 項目 | 仕様 |
|------|------|
| MCU | ATtiny10（SOT-23パッケージ） |
| 筐体 | 12mm × 12mm × 4mm の極小ドングル |
| 接続 | 拡張コネクタ下列4ピン（GND / 5V / SDA / SCL） |
| 演出 | NeoPixel互換シリアルRGB LED × 3粒 |

詳細は別途「ラテカナノ目論見書」参照。
