# Project Handover Document: ラテカピュータキーボード
**Version 7.0 — 設計思想・アドバンテージ・差分を含む完全版**

---

## 1. プロジェクト概要

1980年代のレトロPC「ラテカピュータ」を現代の技術で復刻する。単なる入力装置ではなく、フリスクサイズの「カートリッジ」を受け入れるマルチメディア・ステーションとして機能する。M5Stack CardputerAdvを参照設計としつつ、電源設計・拡張インターフェースにおいて明確なアドバンテージを持つ独自設計とする。

---

## 2. 設計思想と参照設計との比較

### 参照設計：M5Stack CardputerAdv

CardputerAdvは以下の特徴を持つ優れた参照設計である。

- ESP32-S3を搭載したカード型コンピュータ
- HY2.0-4P GROVEポート（5V電源・方向切替スイッチ付き）
- EXT 2.54-14P拡張バス（UART・I2C・SPI・GPIO・電源）
- 内部昇圧回路により常時5Vを維持

### ラテカピュータキーボードの差分とアドバンテージ

| 項目 | CardputerAdv | ラテカピュータキーボード | 理由・アドバンテージ |
|---|---|---|---|
| GROVE電源 | 5V固定（方向切替のみ） | **5V/3V3切替スイッチ** | M5Stackコミュニティの最大の不満を解消。3.3V専用デバイスを改造なしで接続可能 |
| GROVE信号線保護 | レベルシフタなし | **双方向レベルシフタ内蔵（PCA9306D）** | 未知のデバイス接続時もESP32-S3を保護。CardputerAdvでは5V信号が直接ESP32に届くリスクがある |
| 14ピン電源 | 5V IN / 5V OUT（内部昇圧が前提） | **バッテリー入力(3.7V) / 5V OUT** | ESP32-S3は3.3V動作のため5V INは不要。代わりに外部大容量バッテリーの直結端子を設ける |
| カートリッジスロット | なし | **フリスクサイズカートリッジスロット** | 物理的な「人格の容れ物」を接続する独自機能 |
| Qwiicポート | なし | **Qwiic（STEMMA QT）ポート** | 3.3V I2C専用の安全な拡張ポート |
| 内部電源電圧 | 5V（昇圧回路内蔵） | 3.3V（ESP32-S3ネイティブ） | 昇圧回路を省くことでコスト・スペース・電力効率を改善 |

## 3. ハードウェア構成

### メインMCU
- **Waveshare ESP32-S3 Touch LCD 2inch**
  - I2Cマスターとして動作（Qwiic経由）
  - MX1.25コネクタ経由でLiPoバッテリーに接続
  - 充放電管理IC：ETA6098（昇圧機能なし）
  - 3.3V LDO：RT9166A-15PXL（充電ICとは別物）

### Sub MCU（2台構成）

| | UIAPduino-A | UIAPduino-B |
|---|---|---|
| **役割** | I2Cスレーブ・HID担当 | キースキャン専任 |
| **USB使用** | あり（HID） | 書き込み時のみ |
| **電源** | XCL103 OUT(5V) → RAW | XCL103 OUT(5V) → RAW |
| **ESP32-S3との接続** | Qwiic I2C（スレーブ） | なし（直接接続なし） |

UIAPduino2台構成の理由：CH32V003の使用可能GPIOが実質11〜12本に限られるため、1台では5行×11列のキーマトリクス（16本必要）を賄えない。UIAPduino-BはMCP23017をI2Cマスターとして制御してキースキャンを行い、UIAPduino-AはESP32-S3のI2Cスレーブとして結果を転送する役割分担とした。

### IOエキスパンダ
- **Adafruit MCP23017 STEMMA QT版**（43×18mm）
  - UIAPduino-BのI2Cマスター配下（シルク3=SDA、シルク4=SCL）
  - GPA0〜7：ROW×5 + COL×3
  - GPB0〜7：COL×9
  - 合計16本 → 5行×11列キーマトリクス

MCP23017を選んだ理由：競合のPCF8575は「擬似双方向」I/OのためROW線をHIGHからLOWに確実に駆動する能力が弱く、キースキャンに不向き。MCP23017は完全なプッシュプル出力と内蔵プルアップを持ち、ROW/COLの明確な方向制御が可能。

### キー・スイッチ構成

| 種類 | 個数 | 接続先 | 備考 |
|---|---|---|---|
| 通常キー | 5×11列＝55個 | MCP23017経由 | キーマトリクス |
| 12列目キー | 2個（並列） | UIAPduino-B 左列シルク5 | 同機能・並列接続で1本 |
| モード切替スイッチ | 2個（各3モード） | UIAPduino-B 右列シルクA0,7,9,8 | 2本×2で3状態判別 |

ダイオードなし設計の理由：ラテカピュータはテキスト入力が主用途であり、多キー同時押しの頻度が低い。ゴーストキー対策はソフトウェアデバウンス処理で対応する。

### 入力デバイス

| デバイス | 型番 | 接続方法 | I2Cアドレス | 状態 |
|---|---|---|---|---|
| Adafruit Mini I2C Gamepad | #5743 | Qwiic | 0x50（可変） | 到着待ち |
| Playing With Fusion D-Pad | IFB-40003 | Qwiic | 要確認 | 未発注 |
| アナログジョイスティック | ssci.to/2892+2893 | UIAPduino-A直結 | — | 手持ち |

アナログジョイスティックの選定理由：ssci.to/2892はSW（押し込みスイッチ）非搭載のため接続は4本（VCC・GND・VRx・VRy）のみ。UIAPduino-AのADCピン（A2/A3）で読み取る。

### 音声システム

音声信号の流れ：

```
ESP32-S3（I2S出力 IO33/IO34/IO35）
    │ BCLK / WSEL / DIN（MCLKは不要）
    │
PCM5122 DAC（Adafruit #6421）← I2C/SPI設定可
    │ アナログステレオ出力
    │
ボリューム（10kΩ B 2連・物理つまみ RK10J12E002L）
    │
TPA2016アンプ（Adafruit #1712）← I2CでAGC・ゲイン制御
    │
スピーカー × 2（Amazon B098BK7MG2）
```

DAC（PCM5122）を選んだ理由：ESP32-S3のI2S出力をそのままアンプに繋ぐことはできない。PCM5122はMCLK不要でI2SをアナログステレオLに変換する。TPA2016はI2CでAGCとゲインを制御でき、起動時に毎回設定が必要（不揮発保存なし）。

## 4. 外部インターフェース

### インターフェース配置（キーボード上部側面）

```
[左側面]                    [上部側面]                    [右側面]
UIAPduino-A USB            GROVE端子    Qwiic端子           ESP32-S3 USB-C（延長）
UIAPduino系パワースイッチ   フリスクスロット右・ESP32左    ESP32系パワースイッチ
                           5V/3V3切替スイッチ              ボリュームつまみ
                                                           ESP32リセット・ブートボタン（延長）
```

### Qwiicポート（3.3V I2C専用）

- **役割：** 小型センサーの数珠つなぎ用。ESP32-S3がI2Cマスター。
- **電圧：** 3.3V固定。レベルシフタ不要。
- **バス上のデバイス：**

| デバイス | I2Cアドレス |
|---|---|
| UIAPduino-A（スレーブ） | 要設定 |
| Adafruit Mini Gamepad | 0x50（可変） |
| Playing With Fusion D-Pad | 要確認 |
| SparkFun Qwiic Buzzer | 0x34（可変） |
| OLED SSD1306 | 0x3C |

### GROVEポート（HY2.0-4P）

- **役割：** M5Stack Unitエコシステムとの互換。5V系デバイスも安全に接続。
- **設計思想とアドバンテージ：**

M5StackのGROVEは電源が5V固定で信号線にレベルシフタがない。そのため3.3V専用デバイスを接続する際に改造が必要で、コミュニティで長年不満が上がっていた。ラテカピュータキーボードは以下で解決する：

```
ESP32-S3（3.3V）
    │
    PCA9306D（双方向レベルシフタ）← 信号線を3.3V/5V双方向変換
    │
GROVE端子
    ├─ VCC：XCL103(5V) or NJM2845(3V3)をスイッチで切替
    ├─ GND
    ├─ SDA/Signal：レベルシフタ経由
    └─ SCL/Signal：レベルシフタ経由
```

- **5V/3V3切替スイッチ：** 隣接して配置。ユーザーが接続デバイスに合わせて選択。
- **信号線保護：** PCA9306D（SOT-23・超小型）で3.3V↔5V双方向変換。

### 拡張コネクタ（EXT 2.54-14P）

CardputerAdvの拡張バスと同じ設計思想。**14ピン・2列（7ピン×2列）**。ミンティア/フリスクサイズのカートリッジをケーブルなしで直接差し込むスロット。

#### ピン配置

| 列 | ピン | 機能 | 引き出し元 | CardputerAdvとの比較 |
|---|---|---|---|---|
| 下列1 | バッテリー入力（3.7V） | 電源入力 | ダイオード経由でMX1.25へ | CardputerAdvは5V IN（内部昇圧前提）。ラテカピュータは3.3V動作のため3.7Vバッテリー直結が合理的 |
| 下列2 | GND | GND | GND共通 | 同じ |
| 下列3 | 5V OUT | 電源出力 | XCL103経由（バッテリーから独立供給） | CardputerAdvは5V OUT。同じ。カートリッジ側がLDOで3V3に降圧して使用 |
| 下列4 | SDA | I2C | IO47（Qwiicバス共用） | 同じ |
| 下列5 | SCL | I2C | IO48（Qwiicバス共用） | 同じ |
| 下列6 | TX | UART | ESP32-S3空きGPIO | CardputerAdvと同じ |
| 下列7 | RX | UART | ESP32-S3空きGPIO | CardputerAdvと同じ |
| 上列1 | GPIO1 | GPIO | MCP23017 GPA1（将来増設） | CardputerAdvと同じ汎用GPIO |
| 上列2 | GPIO2 | GPIO | MCP23017 GPA2（将来増設） | 同上 |
| 上列3 | GPIO3 | GPIO | MCP23017 GPA3（将来増設） | 同上 |
| 上列4 | GPIO4（CS兼用） | GPIO/SPI CS | MCP23017 GPA4（将来増設） | CardputerAdvと同じCS兼用GPIO |
| 上列5 | SCK | SPI | ESP32-S3空きGPIO | 同じ |
| 上列6 | MOSI | SPI | ESP32-S3空きGPIO | 同じ |
| 上列7 | MISO | SPI | ESP32-S3空きGPIO | 同じ |

#### CardputerAdvとの非互換点

1. **物理形状（オスメス）：** CardputerAdvの拡張コネクタはオス（ピンヘッダ）。ラテカピュータキーボードのスロットはメス（ボックスヘッダ）。信号線配列は同じだが直接の相互接続は不可。
2. **電源入力ピン：** CardputerAdvは5V IN（内部昇圧前提）。ラテカピュータはバッテリー入力（3.7V）。

## 5. 電源構成

### 設計思想

ESP32-S3系とUIAPduino系を独立した電源系統とする。これにより：
- 起動順序の自由度が上がる（電源投入順を気にしない）
- 一方の系統に問題が起きても他方に影響しない
- 将来的なスリープ・省電力制御の独立実装が可能

### ESP32-S3系統

```
内蔵LiPoバッテリー（3.7V）
    │
    ① ヒューズ（113955・1.1A）
    │
    ショットキーダイオード（RB160M-30）─┐
                                         ├─ MX1.25端子 → ESP32-S3（ETA6098）
拡張コネクタ バッテリー入力（3.7V）      │
    └─ ショットキーダイオード（RB160M-30）┘
```

### UIAPduino系統（ESP32-S3と独立）

```
LiPoバッテリー（UIAPduino用・3.7V）
    │
    ① ヒューズ（113955・1.1A）
    │
    XCL103 5V昇圧モジュール（秋月115097）
    │ 5V
    ② ヒューズ（115300・0.5A）
    │
    ├─ UIAPduino-A RAW（5V）
    └─ UIAPduino-B RAW（5V）
```

### 拡張コネクタ電源系統（バッテリーから独立供給）

```
LiPoバッテリー（3.7V）
    │
    ├─ XCL103（5V昇圧）────────────────── 拡張コネクタ 5V OUTピン
    │
    └─ NJM2845DL1-33（3V3降圧）──┐
                                   SP2Tスイッチ → GROVE VCCピン（5V or 3V3切替）
    XCL103（5V昇圧）──────────────┘
```

拡張コネクタの5V OUTはバッテリーから独立したXCL103で供給する。ESP32-S3の3V3ラインは使用しない。これによりカートリッジへの給電がESP32-S3の動作状態に依存しない。

## 7. A-B間通信

| 信号 | A側シルク | B側シルク | 方向 |
|---|---|---|---|
| UART（キースキャン・イベント） | RX PD6（左列2番目） | TX PD5（左列1番目） | B → A |
| GPIO（準備完了通知・汎用） | 10（右列12番目）入力 | 10（右列12番目）出力 | B → A |

SoftwareSerialはCH32V003では動作しないため、ハードウェアUARTを使用する。UIAPduino-BはUSB不使用のためTX(PD5)をUARTに使用可能。UIAPduino-AのTX(PD5)はUSB DPUに使用されており使用不可。

## 10. 起動シーケンス

### 電源投入（順序自由）

```
【UIAPduino系 電源ON】

1. UIAPduino-A 起動
       → I2C初期化完了
       → シルク10（GPIO）をHIGH（準備完了通知）

2. UIAPduino-B 起動
       → シルク10（GPIO）がHIGHになるまで待機
       → HIGH確認後、モード切替スイッチの現在状態をUART送信

【ESP32-S3系 電源ON（UIAPduinoと前後しても可）】

3. ESP32-S3 起動
       → QwiicバスをスキャンしUIAPduino-Aを認識

4. UIAPduino-A がESP32-S3起動を検知
       → A0のLOW→HIGH変化を検出
       → GPIO経由でUIAPduino-Bに「初期値再送要求」を送信

5. UIAPduino-B が再送要求を受信
       → モード切替スイッチの現在状態をUART再送信
```

### 通常動作

```
スイッチ変化時 → イベント駆動でUART送信
キースキャン結果 → 随時UART送信
UIAPduino-A → I2C経由でESP32-S3に転送
```

### ESP32-S3オフ時（単体動作）

```
UIAPduino-A A0がHIGH→LOWに変化（ESP32-S3オフ検知）
→ UIAPduino-A単体モードに移行
→ QwiicバスのGamepad・Buzzer・OLEDを直接制御
```

### スリープ・省電力シーケンス

## 13. 注意事項

- UIAPduino-A のシルク TX（PD5）は USB DPU専用。絶対に配線しないこと。
- シルク7・8・9は左右両列に同じピンが出ている（左右共通）。

- ETA6098がバッテリー充電IC。RT9166A-15PXLは3.3V LDO（別物）。
- NJM2845DL1-05（5V版）を誤購入。3V3版（111299）を別途発注すること。

## 14. M5Unified互換レイヤーとハードウェア抽象化方針

### 基本方針

世の中の膨大なM5Stack系ソフトウェア資産を最大限流用するため、ラテカピュータキーボード用のM5Unified互換レイヤー（HAL）を実装する。ハードウェアの独自性はHAL層で吸収し、アプリケーション層はM5系コードをほぼそのまま流用できる環境を目指す。

### ソフトウェアの層構造

```
【アプリケーション層】← M5系コードをそのまま流用
    M5.Speaker.tone(1000, 100);
    M5.Display.print("Hello");
    M5.BtnA.wasPressed();

【HAL層（latecaputer_hal.h）】← ここだけラテカピュータ固有
    ディスプレイ：M5GFXカスタムパネル定義
    音声：I2Sピン指定（IO33/IO34/IO35）
    キー入力：UIAPduino-AをButton_Classでラップ

【ハードウェア層】← 独自アーキテクチャ
    ディスプレイ：Waveshare ESP32-S3付属LCD
    音声：PCM5122 DAC → TPA2016アンプ
    キー入力：UIAPduino-A（I2Cスレーブ）経由
    GROVE：双方向レベルシフタ付き（M5より高機能）
```

### 各コンポーネントの移植性

| コンポーネント | M5系との差分 | HAL実装方針 | 難易度 |
|---|---|---|---|
| ディスプレイ | Waveshare固有LCD | M5GFXカスタムパネル定義として登録 | 低〜中 |
| 音声 | PCM5122+TPA2016（M5はES8311等） | M5.Speaker cfgでI2Sピン（IO33/34/35）指定。コーデック初期化のみ独自実装 | 低 |
| キー入力 | UIAPduino-A I2Cスレーブ経由 | Button_Classを継承しUIAPduinoポーリング結果をwasPressed()等にマッピング | 中 |
| GROVE | 双方向レベルシフタ付き・電圧切替 | M5のPort.Aと信号線互換。むしろ高機能で互換性に問題なし | なし |
| Qwiic | M5のPort.Aと同等 | そのまま互換 | なし |
| 拡張コネクタ | 独自（M5 Cap互換） | UART/SPI/I2CはM5系と同じプロトコル。ピン番号のみ差分 | 低 |

### M5Unified設定（latecaputer_hal.h）の概要

```cpp
#pragma once
#include <M5Unified.h>

inline void latecaputer_begin() {
    auto cfg = M5.config();

    // Qwiic I2C（外部）
    cfg.external_iic.pin_sda = 47;
    cfg.external_iic.pin_scl = 48;

    // I2S（PCM5122 DAC）
    cfg.speaker.pin_bck      = 33;
    cfg.speaker.pin_ws       = 34;
    cfg.speaker.pin_data_out = 35;
    cfg.speaker.stereo       = true;

    M5.begin(cfg);

    // PCM5122固有の初期化（I2C経由）
    // TPA2016固有の初期化（I2C経由）
}

// UIAPduinoキー入力をM5系Button_Classにマッピング
class LatecaputerKey : public m5::Button_Class {
public:
    bool wasPressed() override {
        return UIAPduino.getKey() == _keycode;
    }
    // ...
};
```

### M5Unified互換の恩恵

- GROVEセンサーのM5Stack向けサンプルがほぼそのまま動く
- M5Stack Unitライブラリが流用できる
- M5系コミュニティの膨大なコード資産を活用できる
- 将来的にM5Unified本体へのラテカピュータボード定義追加PRも検討可能

### 注意事項

M5Unifiedに完全準拠する必要はなく、HAL層でのマッピングのみで十分。ディスプレイドライバ（M5GFX対応可否）とキー入力のラッピングが実装の主な作業となる。

## 15. 活用できるソフトウェアエコシステム

### 既に恩恵を受けているエコシステム

**①Arduino（最大・最古参）**
2005年から続く最も歴史あるエコシステム。ESP32-S3がArduino IDEで開発できるため、Arduinoライブラリの大半がそのまま使える。初心者向けから高度な実装まで膨大な文書・サンプルが蓄積されている。

**②ESP32 Arduino Core**
FastLED・ESPAsyncWebServer・PubSubClient・LVGL・TensorFlow Liteなど人気ライブラリの99%が動作する。Wi-Fi・BLE・I2S・SDカード等のESP32固有機能もArduino APIで使える。ESP32-S3を使っている時点で自動的にこのエコシステムの恩恵を受けている。

**③Adafruit CircuitPython系ライブラリ（Arduino版）**
既に使用しているSTEMMA QT（Qwiic）対応デバイス（MCP23017・Mini Gamepad・Qwiic Buzzer等）はAdafruitのArduinoライブラリがそのまま使える。

### HAL層実装で追加で活用できるエコシステム

**④M5Stack / M5Unified**
M5Unified互換HAL層（latecaputer_hal.h）を実装することで、M5Stack向けに書かれたGROVEセンサードライバ・UIコード・ゲームなどが流用可能になる。M5Stack系コミュニティは日本語情報も豊富。

### エコシステムの整理

| エコシステム | 活用状況 | 主な恩恵 |
|---|---|---|
| Arduino | ✅ 既に恩恵あり | 全ライブラリ流用可・最大の資産 |
| ESP32 Arduino Core | ✅ 既に恩恵あり | Wi-Fi・BLE・I2S・99%のライブラリ |
| Adafruit（Arduino版） | ✅ 既に恩恵あり | STEMMA QT対応デバイスのドライバ |
| M5Stack / M5Unified | △ HAL層実装で活用可 | GROVEセンサー・UIコード・日本語情報 |
| Raspberry Pi / MicroPython | ❌ 対象外 | 別アーキテクチャ |
| CircuitPython | ❌ 対象外 | 別アーキテクチャ |

### 結論

ラテカピュータキーボードはESP32-S3をベースにしている時点で、世界最大級のマイコンエコシステム（Arduino + ESP32）に既に乗っている。M5Unifiedへの対応はそこへのさらなる上乗せであり、日本語情報が豊富なM5Stack系コミュニティの資産も活用できるようになる。

#### 表面・底板・格子足の締結構造

キーマトリクス部は表面と底板を格子足＋M2皿ネジで一体化する。

```
表面（1mm）

    │ 底板側ボス（1.5mm）← 1.5mm噛む
```

### オーバーレイテンプレート板と蓋

オリジナルのラテカピュータキーボードと同様に、キー群を保護するオーバーレイテンプレート板を蓋に固定して持ち運ぶ。

#### テンプレート板の固定

```
蓋裏面
    │ ボス（1mm以上・四隅）
    ↓ はめ込み
テンプレート板（1mm）← 四隅に穴
    ↓ 被せる

```

#### 蓋のスライド固定機構

```
手前 → 奥 方向にスライドして差し込む

キーボード側面：四隅に約2cmの溝を設ける
蓋側：四隅に爪（厚さ1〜1.5mm）← 側面内側に凹ませて蓋幅がキーボード幅を超えない設計

差し込み完了時：
- ESP32-S3部の段差が自然なストッパーになる
- 蓋上面とESP32-S3部が面一に揃う
- 爪が溝に収まり横ズレ防止
```

奥から手前方向へのスライドはESP32-S3部の凸部が干渉するため不可。手前から奥への一方向のみ差し込み可能な設計となり誤挿入防止にもなる。
