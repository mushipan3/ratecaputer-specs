# D基板（日本語入力フロントエンド）ハードウェア仕様書

**Rev. 1.0 — 2026年6月10日**
**対象: CAR-01（UIAPduino版・FRISK筐体）**

---

## 1. 概要

D基板はラテカピュータ（CAR-01規格）の日本語入力フロントエンドを担う専用基板である。ATtiny1604・CH1115 OLED・SKRHAAEジョイスティックの全機能をこの1枚に集約する。日本語入力システムの詳細は `specs/features/japanese_input.md` を参照。

---

## 2. 搭載部品

| 部品 | 型番 | 仕様 | 備考 |
|---|---|---|---|
| MCU | ATtiny1604 | tinyAVR 0-Series・14ピン・16KB Flash・1KB SRAM・20MHz | I2CスレーブかつOLED/ジョイスティック制御 |
| OLED | CH1115 | 0.5インチ・48×88ドット・I2C（0x3C） | 共通I2Cバス接続 |
| ジョイスティック | Alps Alpine SKRHAAE010（秋月 AE-SKRHAAE010-BO） | 4方向＋センタープッシュ・デジタル5ポジション | プルアップ抵抗実装済みDIP変換キット |

---

## 3. ATtiny1604 ピンアサイン

| ピン | 信号名 | 方向 | 接続先 | 備考 |
|---|---|---|---|---|
| PA0 | LOCAL_UPDI_D | INOUT | 共通I2Cバス SDA | UIAPduino PD6（エッジパッド経由）・セルフアップデート用 |
| PA1 | SDA | INOUT | 共通I2Cバス SDA | UIAPduino・ATtiny202と同一バス。CH1115はD基板内でPA1に接続 |
| PA2 | SCL | IN | 共通I2Cバス SCL | 同上 |
| PA3 | JOY_RIGHT | IN | SKRHAAE010 RIGHT端子 | プルアップ済み・LOW=押下 |
| PA4 | JOY_LEFT | IN | SKRHAAE010 LEFT端子 | 同上 |
| PA5 | JOY_UP | IN | SKRHAAE010 UP端子 | 同上 |
| PA6 | JOY_DOWN | IN | SKRHAAE010 DOWN端子 | 同上 |
| PA7 | JOY_CENTER | IN | SKRHAAE010 CENTER端子 | 同上 |
| PB0〜PB3 | （予備） | — | — | 未使用・将来拡張用 |
| PC0〜PC3 | （予備） | — | — | 未使用・将来拡張用 |
| VCC | VCC_3.3V | — | A基板 Pad3（VCC_3V3_OUT） | |
| GND | GND | — | 共通GND | |

**LOCAL_UPDI_Dの兼用制約：**
UIAPduino PD6はYMF825のIRQ_Nとしても使用する。ATtiny1604アップデート中はFM音源を停止すること。UPDIは1対1接続専用。ATtiny202（A基板）はUIAPduino PD0（LOCAL_UPDI_A）で別途接続。同時アップデート不可。

## 4. ジョイスティック仕様

**Alps Alpine SKRHAAE010（秋月電子 AE-SKRHAAE010-BO）**

| 項目 | 仕様 |
|---|---|
| 方式 | デジタル5ポジションスイッチ（アナログ軸なし） |
| 方向数 | 4方向（上・下・左・右）＋センタープッシュ |
| 接点形式 | スイッチ接点（プルアップ時・押下でLOW） |
| 定格 | 50mA / 12V DC |
| 外形 | 7.35×7.5×1.8mm（SMD） |
| 動作寿命 | 200,000サイクル |
| キット付属 | 10kΩプルアップ抵抗実装済み・2.54mmピッチDIP変換基板 |

**ジョイスティック入力ビットフィールド（I2C返答フォーマット・1バイト）：**

```c
#define JOY_UP      0x01  // PA5 LOW
#define JOY_DOWN    0x02  // PA6 LOW
#define JOY_LEFT    0x04  // PA4 LOW
#define JOY_RIGHT   0x08  // PA3 LOW
#define JOY_CENTER  0x10  // PA7 LOW
// bit5〜bit7: 予備（将来拡張）
```

## 5. OLED仕様

CH1115（0.5インチ・48×88ドット）はD基板上に実装し、D基板内でATtiny1604のSDA/SCLと接続する。共通I2Cバスを通じてUIAPduinoからも認識可能（アドレス0x3C）だが、描画はATtiny1604が単独で担当する。

### 5.1 画面レイアウト（横長・高さ48×幅88ドット）

```
┌─────────────────────────────────────────┐  ← 上端
│  確定文字列（9ドット）                     │
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  変換対象ひらがな文（9ドット）              │  区切り対象文字列は反転表示
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  候補文字列群（24ドット）                  │  3行 × 8ドット
├─────────────────────────────────────────┤  ← ライン（1ドット）
│  バッテリー残量（3ドット）                 │
└─────────────────────────────────────────┘  ← 下端
```

### 5.2 バッテリー残量表示

| 項目 | 仕様 |
|---|---|
| 1ブロックのサイズ | 幅8ドット × 高さ3ドット |
| 1ブロックの表す残量 | 10% |
| 満充電（100%）の幅 | 80ドット（8ドット × 10ブロック） |
| 表示位置 | 下端3ドット・左寄せ |
| 端数ブロックの演出 | ゆっくり点滅 |

## 6. 物理設計方針

### 6.1 基板サイズ

| 項目 | 仕様 |
|---|---|
| 基板サイズ | 19mm × 34mm（フリスクケース内収容・SKRHAAE010配置確認後に確定） |
| 基板厚 | 0.8mm |
| 実装方式 | 片面実装 |

### 6.2 ケース内配置

SKRHAAE010のDIP変換キット（2.54mmピッチ）の実装面積が支配的。ジョイスティック操作面をケース窓から露出させる位置に配置する。OLEDもケース表面の窓から露出させる。
